# 金数据 MCP 使用指南

> 一句话介绍：让 AI 助手直接读写你的金数据表单——"搭表单、改字段、查数据、批改数据、看账单"都能用一句中文搞定，不用登录后台。

---

## 一、适用场景

| 角色 | 典型场景 |
| --- | --- |
| 🎯 市场 / 运营 | 活动报名表、问卷；筛选"昨天报名 & 来自上海"的线索 |
| 🛒 销售 / 客服 | 线索登记、批量更新跟进状态、补录数据 |
| 👥 HR / 行政 | 招聘登记、员工信息收集、培训签到 |
| 📊 产品 / 数据 | 调研问卷、多表单汇总分析 |
| 💼 财务 / 老板 | 查账单和付款记录 |

共同特征：**需求已经在文档 / 聊天里写好，不想再切到后台手工操作一遍**。

## 二、分步安装配置

### 前置条件

- 已注册**金数据**账号（[jinshuju.net](https://jinshuju.net)），免费版可体验
- 已安装并登录**支持 MCP 的 AI 客户端**（Claude Desktop、Cursor、WorkBuddy 等均可）

金数据 MCP 支持两种认证方式，任选其一：

- **方式 A · API Key/Secret（HTTP Basic Auth）**：配置一次长期有效，适合自动化场景
- **方式 B · OAuth 2.0**：在客户端里点击授权即可，无需手动管理密钥

MCP 地址（两种方式相同）：**`https://jinshuju.net/mcp`**

---

### 方式 A · API Key/Secret（推荐）

#### Step A1 · 获取 API Key / Secret

1. 登录金数据后台
2. 进入「个人设置 → API / 开发者」
3. 创建并复制 **API Key** 和 **API Secret**（请妥善保管，泄露后建议重置）

#### Step A2 · 生成 Basic 凭证

在终端执行，把 Key 和 Secret 转成 Base64：

```bash
echo -n "YOUR_API_KEY:YOUR_API_SECRET" | base64
```

输出结果（形如 `dXNlcjpwYXNz...`）即下一步要用的 `BASE64_ENCODED_CREDENTIALS`。

#### Step A3 · 在 AI 客户端中配置连接器

连接器信息：

| 字段 | 值 |
| --- | --- |
| **名称** | `jinshuju` 或 `金数据` |
| **URL** | `https://jinshuju.net/mcp` |
| **请求头** | `Authorization: Basic BASE64_ENCODED_CREDENTIALS` |

在客户端的 MCP 配置中粘贴以下片段：

```json
{
  "mcpServers": {
    "jinshuju": {
      "url": "https://jinshuju.net/mcp",
      "headers": {
        "Authorization": "Basic BASE64_ENCODED_CREDENTIALS"
      }
    }
  }
}
```

---

### 方式 B · OAuth 2.0

#### Step B1 · 添加金数据连接器

1. 打开客户端的 **连接器 / Connectors** 管理入口
2. 选择 **添加连接器（Add custom connector）**
3. 填写：
   - **Name**：`金数据` 或 `Jinshuju`
   - **URL**：`https://jinshuju.net/mcp`
4. 点击 **Add**

#### Step B2 · 完成授权

1. 客户端会弹出金数据登录页
2. 用账号登录（手机号 / 邮箱 / 扫码均可）
3. 在授权页点 **允许**
4. 回到客户端看到连接器状态 ✅ 即完成

> OAuth 凭证由客户端托管，token 过期会自动刷新；若长期未使用或账号改密，按 Step B2 重新授权即可。

---

### Step · 首次验证（两种方式通用）

在对话框发送：

> 列一下我金数据里最近 5 个表单

若能返回表单列表，说明全流程打通。🎉

## 三、效果展示

### 案例 1｜一句话搭表单

**输入：**

> 帮我建一个"2026 春季发布会报名表"，字段：姓名（必填）、手机号（必填、校验）、公司、职位、参会城市（下拉：北京/上海/深圳/线上）、感兴趣议题（多选）、备注。主题用蓝色简洁风格。

**对比：**

| | 手工搭建 | MCP 一句话 |
| --- | --- | --- |

## 四、常见误区 & Tips

### ⚠️ 常见误区

1. **"要把数据传给 AI"才能分析** —— 不需要。AI 通过 MCP 拉数据，**数据不离开金数据**。
2. **"改错了没法恢复"** —— 批量操作前 AI 会让你确认；单条修改有历史记录可查。
3. **"用户说中文字段名 AI 就能直接写进去"** —— 不行。金数据 API 的 entry payload 键是每个字段的 `api_code`（如 `field_1`、`field_mobile`），不是中文 label。AI 每次都会先 `get_form` 拿映射再写，这是正常流程，不用人工转换——你说中文就行。
4. **"连接显示 ✅ 但工具不响应"** —— 多半是 API Key / Secret 被重置或凭证过期，重新生成 Basic 凭证并替换配置即可。
5. **"一次只能做一件事"** —— 一条 prompt 可串多个工具（复制表单 → 改标题 → 换主题 → 统计数据）。
6. **"数据量大也能一次拉回来"** —— `list_entries` 单次上限 50 条，分页靠游标。但可以用 `filters` 把字段值条件下推到数据库（等值、范围、子串、是否为空都行），先把命中集打小再翻页，比拉全量快几个数量级。几千条全量记录确实会分页拉取，耗时随数据量增长。

### 💡 使用 Tips

- **✨ 先「看一眼」再动手**：批量操作前让 AI 先 `get_form` 确认字段，准确率更高
- **✨ 带时间 + 数量**：避免一次拉几千条，加"最近 7 天""前 50 条"更稳
- **✨ 主动要求脱敏**：「手机号打星」「只显示姓氏」AI 会自动处理
- **✨ 两步法做报表**：第一步 MCP 拉数据，第二步让 AI 用代码做透视图
- **✨ 把常用操作固化成 prompt 模板**：例如「每周一自动拉上周新增数据摘要」

### 🆘 遇到问题

| 现象 | 解决办法 |
| --- | --- |
| 一直转圈连不上 | 刷新重试；仍失败 → 删除连接器重新添加 |
| 字段类型建不出来 | 38 种字段类型已支持（含矩阵 / 商品 / 公式 / 关联表单等）；部分高级字段需付费套餐，错误提示会直接显示需要的套餐 |
| 提示"表单不存在" | 核对 form_token；确认是同一个金数据账号 |
| 操作被拒绝 | 该表单是协作表单且你无编辑权限，联系所有者 |

## 五、安全与权限

- MCP 所有操作都按当前凭证所属账号的权限执行——看不到你无权访问的数据
- 重要操作（批量删除、覆盖字段）AI 会要求显式确认
- **Basic 方式**：凭证以 Base64 形式保存在客户端配置；若泄露，到「个人设置 → API / 开发者」**重置 Secret**，旧凭证立即失效；本地 `mcp.json` 请勿提交到公共仓库
- **OAuth 方式**：凭证由客户端托管，可随时在金数据「设置 → 授权管理」中撤销对应客户端的授权
